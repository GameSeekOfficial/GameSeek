# 🤝 Contributing to GameSeek

> *Thank you for your interest in contributing to GameSeek!*  
> *Please read this guide carefully before you start.*

-----

## 🐛 Reporting Bugs & Issues

Found a bug or something not working as expected?

**Please do not open a GitHub Issue directly.**

Instead, report it through our official support page:

🔗 **[gameseekapp.com/support/index.html](https://gameseekapp.com/support/index.html)**

Our team will review every report and respond as soon as possible.

-----

## 📋 Developer Standards

To keep the project clean, organized, and maintainable, all contributors must follow these rules:

### 🔒 Always Create a Backup

Before making any changes — always create a backup of the current state.  
Never work directly on something without having a safe restore point.

### 📝 Document Everything

Write down what you are doing, what you changed, and why.  
Every meaningful change must be documented — no undocumented modifications.

### 🧹 Keep It Clean

- Write readable, well-structured code
- Remove unused code, comments, and files before submitting
- Follow the existing code style of the project

### 🔍 Test Before Submitting

Test your changes thoroughly before submitting anything.  
Do not submit untested or broken work.

### 💬 Communicate

If you are unsure about something — ask first, change later.  
Coordinate with the team before making major structural changes.
